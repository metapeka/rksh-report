Плагин @rksh/blocks: кастомные блоки Portable Text

Назначение

Плагин добавляет 7 типов кастомных блоков в редактор Portable Text EmDash, чтобы быстро создавать структурированные страницы без ручного форматирования.

Проблема, которую решает: Без плагина каждый блок (hero, карточки преимуществ, таблица расписания) пришлось бы собирать из параграфов, заголовков и таблиц вручную. Это долго и непоследовательно. Плагин даёт готовые шаблоны с предопределёнными полями.

Структура плагина

plugins/rksh-blocks/
├── package.json              # Мета: name="@rksh/blocks", exports
├── src/
│   ├── index.ts              # PluginDescriptor (build-time)
│   │   └── возвращает: { id, version, entrypoint, componentsEntry }
│   ├── sandbox-entry.ts      # createPlugin() — runtime определение
│   │   └── admin.portableTextBlocks: [ 7 типов блоков ]
│   └── astro/
│       ├── index.ts          # export const blockComponents = { ... }
│       ├── Hero.astro        # Компонент rksh.hero
│       ├── Features.astro    # rksh.features
│       ├── Principles.astro  # rksh.principles
│       ├── Advantages.astro  # rksh.advantages
│       ├── Schedule.astro    # rksh.schedule
│       ├── Teachers.astro    # rksh.teachers
│       └── Cta.astro         # rksh.cta

Разделение контекстов (важно!)

Build-time (src/index.ts):

  • Импортится в astro.config.mjs через rkshBlocks()
  • Возвращает PluginDescriptor — только метаданные, без side-effects
  • Не может использовать definePlugin(), API EmDash

Runtime (src/sandbox-entry.ts):

  • Загружается на сервере при обработке запроса
  • Экспортирует именованную функцию createPlugin(options)
  • Вызывает definePlugin({...}) — тут уже можно использовать hooks, storage, admin config
  • Именно здесь объявляются admin.portableTextBlocks

Ошибка createPlugin is not a function возникает, когда:

  • sandbox-entry.ts экспортирует export default definePlugin({...})
  • EmDash ищет createPlugin как named export, а не default
  • Решение: всегда export function createPlugin() { return definePlugin({...}) }

Типы блоков (полная спецификация)

1. rksh.hero — Hero-секция

Purpose: Визуальный заголовок страницы с фоном, подзаголовком и CTA-кнопкой.

Поля:

Поле Тип Обязательное Пример
title string ✅ «Частная школа РКШ — Москва, Куркино»
subtitle string ❌ «Идет набор на 2025-2026 учебный год»
cta_text string ❌ «Получить презентацию»
cta_url string ❌ /kontakty или #presentation

Компонент: plugins/rksh-blocks/src/astro/Hero.astro

Стили:

  • Фон: var(--color-primary) (тёмно-синий)
  • Текст: белый
  • Кнопка: var(--color-accent-warm) (тёплый жёлто-коричневый)
  • Отступы: 4xl сверху/снизу

Использование:

  • Главная страница
  • Все основные страницы (О школе, Цены, Контакты...)

Notes:

  • Если cta_text пустой — кнопка не отображается
  • cta_url без title — ссылка не появляется

2. rksh.features — Карточки особенностей

Purpose: Сетка из 6 карточек с номерами, заголовками и описаниями. Для «6 преимуществ», «6 особенностей программы».

Поля:

Поле Тип Обязательное Описание
title string ❌ Заголовок секции (например «Почему выбирают нас»)
f1_title string ✅ Заголовок карточки 1
f1_desc string ❌ Описание карточки 1
… … … … (аналогично f2–f6)
f6_title string ✅ Заголовок карточки 6
f6_desc string ❌ Описание карточки 6

Компонент: Features.astro

Логика компонента:

const features = [
  { title: f1_title, desc: f1_desc },
  ...
].filter(f => f.title); // автоматически убирает пустые

Стили:

  • Сетка: grid-template-columns: repeat(auto-fill, minmax(300px, 1fr)) → 3×2 на десктопе
  • Карточка: фон surface, борader primary-line, hover → shadow + accent border
  • Номер: крупный (4xl), цвет primary-soft (светло-синий)

Использование:

  • Главная: «6 ключевых преимуществ»
  • О школе: «6 особенностей нашей программы»

Ограничение: Жёстко 6 слотов. Чтобы добавить 7-ю карточку — нужно менять код плагина и компонент.

3. rksh.principles — Принципы обучения

Purpose: Нумерованный список принципов образовательной философии школы. Визуально похож на features, но с другим стилем (может использоваться для «Миссия», «Наши принципы»).

Поля:

Поле Тип Обязательное Пример
title string ❌ «Наши принципы»
p1_title – p6_title string ✅ «Без домашних заданий», «Маленькие классы»...
p1_desc – p6_desc string ❌ Описание принципа

Компонент: Principles.astro (реализация аналогична Features.astro, другие CSS-классы)

Стили:

  • Номер: font-size: 4xl, opacity: 0.5
  • Карточка: возможно, без бордера, просто разделитель

Использование:

  • Страница «О школе» — секция «Принципы обучения»
  • Главная — «Миссия» (но там уже есть advantages)

4. rksh.advantages — Преимущества (numbered list)

Purpose: Нумерованный список преимуществ, визуально отличим от features и principles. Может использоваться для блока «Чем мы лучше обычной школы».

Поля:

Поле Тип Обязательное
title string ✅
a1_title – a6_title string ✅
a1_desc – a6_desc string ❌

Компонент: Advantages.astro

Стили: Отличаются от features:

  • Может быть без карточек, просто numbered list
  • Номера выделены жирно, возможно в круге

Пример контента (из content-summary.md):

  1. «Классы до 15 человек»
  2. «Работаем в удобном режиме с 8:00 до 19:00»
  3. «Высокое качество образования по системе РКШ» ... (6 пунктов)

5. rksh.schedule — Таблица расписания

Purpose: Таблица занятий для одного класса/группы.

Поля:

Поле Тип Обязательное Пример
title string ❌ «Расписание 1 класса»
class_name string ✅ «1 класс»
time_range string ✅ «8:00 — 14:00»
mon string ❌ «Математика, Русский, Физкультура»
tue string ❌ …
wed string ❌ …
thu string ❌ …
fri string ❌ …

Компонент: Schedule.astro

Рендер:

<table>
  <thead><tr><th>Пн</th><th>8:00–9:00</th>...</tr></thead>
  <tbody>
    <tr><td>Понедельник</td><td>mon content</td></tr>
    ...
  </tbody>
</table>

Ограничение: Один блок = один класс. Для 10 классов нужно 10 блоков на странице (неудобно). Планируется заменить на связь с коллекцией schedule.

6. rksh.teachers — Карточки преподавателей

Purpose: Сетка из 1–3 карточек педагогов (сейчас ограничено 3).

Поля:

Поле Тип Обязательное Пример
title string ❌ «Наши преподаватели»
t1_name – t3_name string ✅ «Иванов Иван Иванович»
t1_role – t3_role string ✅ «Учитель математики»
t1_desc – t3_desc string ❌ «Стаж 15 лет...»
t1_photo – t3_photo string (URL) ❌ https://.../ivanov.jpg

Компонент: Teachers.astro

Рендер:

<div class="teachers-grid">
  <article class="teacher-card">
    <img src={t1_photo} alt={t1_name} />
    <h3>{t1_name}</h3>
    <p>{t1_role}</p>
    <p>{t1_desc}</p>
  </article>
  ...
</div>

Ограничение: Жёстко 3 карточки. Нельзя добавить 4-го педагога без изменения кода.

Планы: Переделать на связь с коллекцией teachers → тогда блок будет запрашивать всех педагогов из БД.

7. rksh.cta — CTA-секция

Purpose: Заключительный блок с призывом к действию.

Поля:

Поле Тип Обязательное Пример
title string ✅ «Остались вопросы?»
description string ❌ «Оставьте заявку, и мы вам перезвоним»
button_text string ✅ «Оставить заявку»
button_url string ✅ /kontakty

Компонент: Cta.astro

Стили:

  • Центрированный блок
  • Фон: var(--color-surface-alt) (светло-синий)
  • Кнопка: акцентный цвет
  • Отступы: 3xl

Редактирование блоков в админке

Механизм Block Kit

EmDash использует Block Kit — форму на основе Slack-like полей. Каждый блок в admin.portableTextBlocks описывает:

{
  type: "rksh.hero",
  label: "Hero-секция",
  icon: "link", // или "image", "video", "code"
  description: "...",
  fields: [
    { type: "text_input", action_id: "title", label: "Заголовок" },
    { type: "text_input", action_id: "subtitle", label: "Подзаголовок" },
    { type: "text_input", action_id: "cta_text", label: "Текст кнопки" },
    { type: "text_input", action_id: "cta_url", label: "Ссылка кнопки" },
  ]
}

Типы полей Block Kit:

  • text_input — однострочное текстовое поле
  • textarea — многострочное
  • select — выпадающий список (нужен options: [{value, label}])
  • url — валидация URL
  • image_upload — загрузка изображения (пока не используется)

Как добавлять валидацию

Сейчас все поля необязательные. Чтобы сделать обязательным:

{ type: "text_input", action_id: "title", label: "Заголовок", required: true }

EmDash не пропустит сохранение, если поле пустое.

Как добавить的选择 (dropdown)

{ type: "select", action_id: "cta_style", label: "Стиль кнопки", 
  options: [
    { value: "primary", label: "Основной" },
    { value: "secondary", label: "Вторичный" }
  ]
}

В компоненте: const { cta_style } = Astro.props.node;

Расширение плагина

Сценарий: Нужно 8 карточек в features

Шаги:

  1. Обновить sandbox-entry.ts — добавить поля f7_title, f7_desc, f8_title, f8_desc в fields массиве для rksh.features

  2. Обновить компонент Features.astro:

const { title, f1_title, f1_desc, ..., f8_title, f8_desc } = Astro.props.node;
const features = [
  { title: f1_title, desc: f1_desc },
  ...
  { title: f8_title, desc: f8_desc },
].filter(f => f.title);

Или лучше преобразовать в массив:

const featureCount = 8;
const features = Array.from({ length: featureCount }, (_, i) => ({
  title: Astro.props.node[`f${i+1}_title`],
  desc: Astro.props.node[`f${i+1}_desc`],
})).filter(f => f.title);

  1. Обновить CSS, если нужно изменить сетку (с 6 на 8 карточек)

  2. Перезапустить dev-сервер (плагин пересобирается)

  3. Обновить документацию (02-blocks-plugin.md)

Сценарий: Подключить изображения в teachers

Сейчас: t1_photo — строка с URL (текстовое поле).
Хотим: загружать через медиа-библиотеку.

Решение:

  1. В sandbox-entry.ts изменить тип поля:
{ type: "image_upload", action_id: "t1_photo", label: "Фото 1" }

Или оставить text_input, но в компоненте использовать <Image image={...} /> из emdash/ui.

  1. В Teachers.astro:
import { Image } from "emdash/ui";
<Image image={t1_photo} alt={t1_name} />

image должен быть объектом {src, alt}. Если в админке загружено через медиа-библиотеку, EmDash сам подставит правильный объект.

Стилизация

Каждый компонент содержит <style> внутри. Это scoped styles (Astro по умолчанию).

Глобальные переменные: компоненты используют var(--color-*), var(--spacing-*) из src/styles/tokens.css.

Изменение стиля всех блоков:

  1. Отредактировать tokens.css (палитра, шрифты, spacing)
  2. Или переписать CSS внутри каждого .astro файла

Рекомендация: вынести общие классы (.rksh-section, .rksh-container) в global.css, а в компонентах использовать только @apply или уточняющие селекторы.

Отладка

Блоки не отображаются на фронте

  1. Проверить, что PortableText импортирован:
import { PortableText } from "emdash/ui";
<PortableText value={page.data.content} />

  1. Убедиться, что blockComponents экспортированы в plugins/rksh-blocks/src/astro/index.ts

  2. Проверить консоль браузера: ошибки импорта компонентов?

  3. Проверить серверные логи: createPlugin is not a function → плагин не загружен

Блоки не видны в админке (не появляются в / меню)

  1. Перезапустить dev-сервер (плагин подгружается при старте)
  2. Проверить astro.config.mjs — есть ли plugins: [rkshBlocks()]
  3. Убедиться, что в sandbox-entry.ts блоки объявлены под admin.portableTextBlocks
  4. Проверить версии emdash и @rksh/blocks — совместимость по peerDependencies

Изменения в коде не применяются

  • Dev-сервер должен перезапуститься (Vite HMR может не подхватить плагин)
  • Полностью остановить и запустить заново
  • Очистить кэш: rm -rf .astro/ node_modules/.vite
