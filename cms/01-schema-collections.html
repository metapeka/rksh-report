Связи между коллекциями

// teachers → courses
teacher.data.courses = [
  { id: "course-1", collection: "courses_preschool" },
  { id: "course-2", collection: "courses_school" }
];

// advantages → page (optional)
advantage.data.page = { id: "o-shkole", collection: "pages" };

// schedule → school_levels
schedule.data.grade_ref = { id: "1-klass", collection: "school_levels" };

// reviews → teachers (who taught)
reviews.data.teacher = { id: "ivanov", collection: "teachers" };

// prices → courses
price.data.program = { id: "podgotovka-4-7", collection: "courses_preschool" };

Миграция seed.json

Текущий seed/seed.json содержит только pages, posts, settings. Нужно добавить все новые коллекции:

{
  "collections": [
    { "slug": "teachers", "label": "Преподаватели", "fields": [...] },
    { "slug": "schedule", "label": "Расписание", "fields": [...] },
    ...
  ]
}

Порядок внесения:

  1. Определить все поля
  2. Записать в seed.json
  3. Запустить npx emdash dev (применит миграцию)
  4. Заполнить контент через админку

Дополнительные коллекции (опционально)

  • applications — заявки с форм (хранилище)
  • subscribers — подписчики на рассылку
  • events — события школы (экскурсии, дни открытых дверей)

FAQ

Q: Можно ли использовать связи many-to-many?
A: Да, через массив reference: [{id, collection}, ...]

Q: Как удалить поле из коллекции?
A: В seed.json удалить поле → пересоздать БД (dev) или написать миграцию (prod). Данные поля потеряются.

Q: Можно ли изменить тип поля (string → number)?
A: Да, но существующие значения конвертировать вручную через SQL.

Q: Как добавить поле required: true?
A: В seed.json добавить "required": true в поле. Для существующих записей пустые значения станут invalid.

# С/jsonb поле content
