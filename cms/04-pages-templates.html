Шаблоны страниц и роутинг

Обзор системы маршрутизации

Astro использует файл-базированный роутинг:

src/pages/
├── index.astro              →  /
├── posts.astro              →  /news/  (список)
├── posts/[id].astro         →  /posts/[id]  (детальная)
├── pages/[slug].astro       →  /[slug]  (любая статическая страница)
└── (другие маршруты)...

Важно: Страницы из коллекции pages (EmDash) обслуживаются одним шаблоном pages/[slug].astro. Это динамический SSR-роут — страница генерируется на каждый запрос на основе slug.

Шаблоны Astro

src/layouts/Base.astro

Назначение: базовый layout всех страниц.Подключает:

  • Header (основное меню)
  • SecondMenu (второе меню)
  • <slot /> — контент страницы
  • Footer
  • Скрипт для dropdown-меню (клик вне закрывает)

Props:

interface Props {
  title?: string; // defaults to "РКШ — Русская Классическая Школа"
}

Использование:

<Base title="О школе">
  <h1>Заголовок страницы</h1>
  <p>Контент...</p>
</Base>

src/pages/index.astro

URL: /
Тип: SSR page
Логика:

  1. Загружает последние 10 новостей из posts (getEmDashCollection)
  2. Отображает их списком под приветствием
  3. Но основной контент главной (hero, features, advantages) должен управляться через запись в pages с slug=home

Текущее состояние:

  • Есть статический HTML с заголовком «Добро пожаловать!»
  • Нет интеграции с блоками @rksh/blocks
  • Необходимо: заменить на <PortableText value={homePage.data.content} />

План:

---
import Base from "../layouts/Base.astro";
import { getEmDashEntry } from "emdash";

const { entry: home, cacheHint } = await getEmDashEntry("pages", "home");
Astro.cache.set(cacheHint);
---

<Base title={home.data.title}>
  <PortableText value={home.data.content} />
</Base>

src/pages/pages/[slug].astro

URL: /[slug] (любой slug из pages)
Тип: Dynamic SSR route
Параметр: slug (из URL)

Логика:

export async function getStaticPaths() {
  const { entries } = await getEmDashCollection("pages", { limit: 100 });
  return entries.map(entry => ({
    params: { slug: entry.id },
    props: { entry }
  }));
}

// При запросе /o-shkole/:
const { entry: page, cacheHint } = await getEmDashEntry("pages", slug);
Astro.cache.set(cacheHint);

Важно: getStaticPaths здесь не генерирует статически — это просто список всех slugs, которые Astro зарегистрирует как routable. Фактические данные загружаются на каждый запрос (SSR).

Рендер:

<Base title={page.data.title}>
  <article>
    <h1>{page.data.title}</h1>
    <PortableText value={page.data.content} />
  </article>
</Base>

src/pages/posts.astro

Показ: заголовок новости, excerpt (если есть), дата

Динамические vs Статические страницы

Характеристика Статические (SSG) Динамические (SSR)
Генерация Билд-тайм Рантайм (на каждый запрос)
Кэш CDN/файлы In-memory/Redis (cacheHint)
Подходит для Маркетинг-лендинги, docs CMS-контент, блоги
getStaticPaths ✅ Генерирует файлы ❌ Не используем для CMS

Правило EmDash: ВСЕ страницы с контентом из БД — SSR только. Никакого getStaticPaths для CMS-записей.

Slug (URL-идентификатор)

Что такое slug?

  • human-readable идентификатор в URL: /o-shkole/, prepodavateli/
  • генерируется автоматически из title (транслитерация, lowercase, hyphens)
  • можно редактировать в админке
  • должен быть уникальным в пределах коллекции

Пример:

title: "О школе" → slug: "o-shkole"
title: "Наши преподаватели" → slug: "nashi-prepodavateli" (но обычно `prepodavateli`)

Важно: entry.id возвращает slug. В getEmDashEntry(collection, id) второй аргумент — это slug, а не ULID.

Параметры маршрута

pages/[slug].astro:

const { slug } = Astro.params; // из URL

Примеры:

  • /o-shkole/ → slug = "o-shkole"
  • /czeny/ → slug = "czeny"

404 сторона

Не создана. Нужно создать src/pages/[...slug].astro или 404.astro.

Рекомендация:

---
import Base from "../layouts/Base.astro";
---
<Base title="Страница не найдена">
  <h1>404 — Страница не найдена</h1>
  <p>Запрошенная страница не существует.</p>
  <a href="/">Вернуться на главную</a>
</Base>

Мета-теги страниц

Как задать title/description для каждой страницы?

  1. В Base.astro:
<head>
  <title>{title}</title>
  <!-- EmDash добавляет SEO через EmDashHead? -->
</head>
  1. Использовать getSeoMeta() из emdash:
---
import { getSeoMeta } from "emdash";
const seo = await getSeoMeta({ title: page.data.title, description: page.data.excerpt });
---
<Base title={seo.title}>
  <head>
    <EmDashHead />
  </head>
  ...
</Base>

Пока упрощённо: передаём title в Base. Для SEO нужен отдельный блок или поле в pages.

Резервные варианты (fallback)

Если страница не найдена (getEmDashEntry вернул null):

const { entry, cacheHint } = await getEmDashEntry("pages", slug);
if (!entry) {
  return Astro.redirect("/404");
  // или рендерить 404 прямо здесь
}

Кэширование страниц

Обязательно на каждой странице:

Astro.cache.set(cacheHint);

Без этого:

  • При редактировании страницы в админке изменения не появятся на фронте
  • Кэш EmDash не будет инвалидироваться

Страницы-коллекции vs одиночные записи

  • posts — коллекция, имеет список (posts.astro) и детальную (posts/[id].astro)
  • pages — каждый slug = отдельная страница, нет списка страниц (кроме главной)

Если нужен список страниц (например, все курсы):

const { entries } = await getEmDashCollection("courses_preschool");

И рендерить в CollectionPage.astro.

Динамические route parameters

Пример: /doshkolniki/podgotovka-k-shkole-ot-4-do-7-let/

Если хотим отдельный шаблон:

src/pages/doshkolniki/[slug].astro

Можно получить slug курса и загружать данные из courses_preschool по нему.

Структура:

const { slug } = Astro.params; // "podgotovka-k-shkole-ot-4-do-7-let"
const { entry: course } = await getEmDashEntry("courses_preschool", slug);

i18n (международность)

Пока: только русский язык (lang="ru" в Base.astro).

Планы: если понадобится английская версия:

  • Создать src/pages/en/ с такими же шаблонами
  • Использовать getEmDashEntry с фильтром по lang (нужно добавить поле lang в коллекции)
  • Или дублировать каждую запись с отдельным slug (o-shkole-en)

Best practices

  1. Всегда Astro.cache.set(cacheHint) после getEmDash* запросов
  2. Никогда не использовать getStaticPaths для CMS-контента
  3. Изображения — <Image image={...} /> из emdash/ui
  4. Ссылки — использовать entry.id (slug), а не entry.data.id (ULID)
  5. Обработка ошибок: проверять if (!entry), рендерить 404
