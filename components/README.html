Библиотека компонентов РКШ

Каталог повторяющихся UI-паттернов проекта. Агент использует его как справочник: вместо «сделай карточку как-нибудь» → «используй Card variant=surface».

Как читать

Файл Что внутри

00-catalog.md Быстрый индекс: номер → название → тип → где используется
01-card.md, 02-section-title.md … Spec отдельного компонента

Архитектурные решения

Пропы, не CSS-классы

Компоненты принимают варианты через проп variant, не через CSS-классы:

✅ <Card variant="elevated">
❌ <Card class="card--elevated">

Причины:

  • TypeScript ловит опечатки (variant="elavated" → ошибка, class="card--elavated" → молча ломается)
  • Агент получает однозначную инструкцию: «Используй Card variant=muted»
  • Редизайн в одном месте: меняешь Card.astro → все страницы обновляются
  • Ограниченный API лучше свободы — агент не придумает несуществующий вариант

Единственная точка правды

Код (src/components/ui/Card.astro) — единственная точка правды для CSS. Spec (docs/components/01-card.md) — справочник для агента, не копипаст-источник. Spec ссылается на .astro-файл. Если расходится — код приоритет.

Showcase-страница

/styleguide — живой превью всех компонентов. Только в dev, в продакшене редиректит на 404.

При добавлении нового компонента — добавить секцию на styleguide.

Нумерация

Диапазон Уровень
01–09 Базовые (атомарные)
10–19 Составные (из базовых + CMS-данные)
20–29 Страничные (уникальные для одной страницы)

Принцип

  1. Не дублируй дизайн-гайд — компонент ссылается на page-specs/01-design-guide.md, описывает только своё
  2. Единственная точка правды — .astro-файл = код, .md-файл = документация для агента
  3. Агент сначала сюда — перед созданием нового UI-паттерна агент смотрит каталог и берёт существующий

Уровни компонентов

Уровень Описание Примеры
Базовый Атомарные, используются везде Card, Button, Badge, SectionTitle, IconList
Составной Собраны из базовых, привязаны к данным CMS ProgramCard, TeacherCard, ReviewCard, SeatsCard
Страничный Уникальный для одной страницы Hero, MissionQuote, EnrollSteps

Процесс добавления нового компонента

  1. Паттерн замечен на 2+ страницах → вынести
  2. Создать src/components/ui/X.astro с пропом variant
  3. Создать docs/components/NN-x.md (следуя 00-guide.md)
  4. Добавить секцию на src/pages/styleguide.astro
  5. Обновить docs/components/00-catalog.md
  6. Заменить inline-код на страницах на <X variant="...">

Структура файлов

  README.md                ← этот файл (入口)
  00-guide.md              ← правила написания spec'ов
  00-catalog.md            ← быстрый индекс
  01-card.md               ← spec Card
  02-section-title.md      ← (следующие)

  Card.astro               ← базовый компонент
  Button.astro             ← (следующие)

src/pages/
  styleguide.astro         ← живой showcase (dev-only)
