Справочник API EmDash

Content API

getEmDashCollection(collection, options)

Получить список записей из коллекции.

Параметры:

interface GetCollectionOptions {
  limit?: number;          // default 20, max 100
  cursor?: string;         // пагинация (ULID)
  orderBy?: Record<string, "asc" | "desc">; // { published_at: "desc" }
  where?: Record<string, any>; // фильтры: { status: "published" }
  include?: string[];      // include related (если есть references)
}

Возвращает:

{
  entries: Entry[];
  nextCursor?: string;     // для следующей страницы
  cacheHint: { ttl: number; tags: string[] }; // кэширование
}

Пример:

const { entries, cacheHint } = await getEmDashCollection("posts", {
  limit: 10,
  orderBy: { published_at: "desc" },
  where: { status: "published" },
});
Astro.cache.set(cacheHint);

getEmDashEntry(collection, id)

Получить одну запись по slug (id = slug, не ULID).

Пример:

const { entry, cacheHint } = await getEmDashEntry("pages", "o-shkole");
Astro.cache.set(cacheHint);

Error: если запись не найдена → entry = null.

search(query, options)

Полнотекстовый поиск по индексируемым полям.

Пример:

const results = await search("школа", {
  collections: ["pages", "posts"],
  limit: 20,
});

Site Features API

getSiteSettings()

Получить настройки сайта из коллекции settings (первая запись).

const settings = await getSiteSettings();
// settings.site_name, settings.logo, settings.phone_primary...

getMenu(name)

Получить меню по имени (primary, secondary, footer).

Returns:

{
  name: string;
  items: MenuItem[];
}
// MenuItem: { label, url, target?, children: [] }

getTerms(taxonomy)

Получить все термины таксономии.

const categories = await getTerms("news_category");
// [{ id, name, slug, description, parent? }]

getTerm(taxonomy, slug)

Один термин по slug.

const newsCategory = await getTerm("news_category", "news");

getEntryTerms(entry, taxonomy)

Получить термины, прикреплённые к записи.

const postTerms = await getEntryTerms(post, "news_category");

getEntriesByTerm(taxonomy, termSlug, options)

Все записи, относящиеся к термину.

const newsPosts = await getEntriesByTerm("news_category", "news", {
  limit: 10,
});

getWidgetArea(name)

Получить виджеты области.

const sidebar = await getWidgetArea("sidebar");
// [{ id, type, data: {...} }]

getSection(name)

Получить секцию (если используется Sections feature).

const hero = await getSection("home_hero");

getSeoMeta(options)

Сгенерировать SEO-теги.

const seo = await getSeoMeta({
  title: "О школе",
  description: "Описание...",
  image: { src: "/img/og.jpg", alt: "..." },
  url: Astro.url,
  type: "website", // или "article"
});
// seo.tags: Array<{ tag: string; attrs: Record<string, string> }>

Bylines API

Билайны — это авторы/сотрудники (Bylines collection).

getBylinesForEntries(entries)

Загрузить всех авторов для списка записей.

const posts = await getEmDashCollection("posts", { limit: 10 });
const bylines = await getBylinesForEntries(posts);
// bylines: { [entryId]: Byline[] }

getEntryBylines(entry)

Авторы конкретной записи.

const postBylines = await getEntryBylines(post);

Storage API (для плагинов)

Внутри definePlugin():

ctx.storage.<collection>

CRUD:

// Put (create/update)
await ctx.storage.items!.put("item-id", { name: "Test" });

// Get
const item = await ctx.storage.items!.get("item-id");
// item = { id, data: {...}, created_at, updated_at }

// Query
const result = await ctx.storage.items!.query({
  filter: { status: "active" },
  orderBy: { created_at: "desc" },
  limit: 10,
  cursor: "...",
});
// result.items, result.nextCursor, result.hasMore

// Delete
await ctx.storage.items!.delete("item-id");

// Count
const count = await ctx.storage.items!.count();

Индексы (задают в PluginDescriptor.storage):

storage: {
  logs: {
    indexes: ["timestamp", "level", ["level", "timestamp"]],
  },
},

KV Store (плагины)

Key-value хранилище для настроек, кэша.

// Set
await ctx.kv.set("setting:key", value, { ttl: 3600 });

// Get
const val = await ctx.kv.get<string>("setting:key");

// Delete
await ctx.kv.delete("setting:key");

// Batch
await ctx.kv.setMany([
  ["key1", "val1"],
  ["key2", "val2"],
]);

Hooks API

Список хуков

Hook When Параметры
plugin:install При установке плагина (event, ctx)
plugin:uninstall При удалении (event, ctx)
content:beforeSave Перед сохранением записи (event, ctx)
content:afterSave После сохранения (event, ctx)
content:afterDelete После удаления (event, ctx)
content:beforePublish Перед публикацией (event, ctx)
content:afterPublish После публикации (event, ctx)
user:afterCreate После создания пользователя (event, ctx)
user:beforeDelete Перед удалением пользователя (event, ctx)
media:afterUpload После загрузки файла (event, ctx)

Пример hook

hooks: {
  "content:afterSave": async (event, ctx) => {
    const { content, collection, isNew } = event;
    ctx.log.info("Content saved", { collection, id: content.id });
    
    // Отправить уведомление в Telegram
    await ctx.fetch("https://api.telegram.org/...", {
      method: "POST",
      body: JSON.stringify({ text: `Новая запись: ${content.data.title}` }),
    });
  },
}

Опции хука:

{
  priority: 100,  // порядок выполнения (меньше = раньше)
  timeout: 5000,  // ms, после чего хук прерывается
}

Routes API (плагины)

API роуты доступны по /_emdash/api/plugins/<plugin-id>/<route>.

routes: {
  status: {
    handler: async (ctx) => {
      const count = await ctx.storage.items!.count();
      return { count, version: "1.0.0" };
    },
    // auth: true // требует авторизации
  },
  
  webhook: {
    method: "POST",
    handler: async (ctx) => {
      const body = await ctx.request.json();
      // обработка
      return { ok: true };
    },
  },
}

Использование:

GET /_emdash/api/plugins/rksh-blocks/status
POST /_emdash/api/plugins/rksh-forms/webhook

Request Context

Внутри route/handler/createPlugin() доступен ctx:

interface PluginContext {
  log: Console;         // логгер
  db: Database;         // доступ к БД (Kysely)
  storage: Storage;     // ctx.storage.<collection>
  kv: KV;              // key-value store
  config: Record<string, any>; // настройки плагина
  request: Request;     // текущий HTTP request (в route)
  fetch: typeof fetch;  // для внешних запросов
  env: Env;            // переменные окружения
}

UI Components (EmDash UI)

PortableText

Рендерит Portable Text content.

import { PortableText } from "emdash/ui";
<PortableText value={page.data.content} />

Кастомные блоки: автоматически подключаются, если плагин экспортирует blockComponents.

Image

Оптимизированный компонент изображения.

import { Image } from "emdash/ui";
<Image 
  image={post.data.featured_image}
  alt={post.data.featured_image.alt || "..."}
  widths={[400, 800, 1200]}
  sizes="(max-width: 768px) 100vw, 800px"
  loading="lazy"
/>

Comments, CommentForm

Комментарии к записям.

import { Comments, CommentForm } from "emdash/ui";
<Comments entryId={post.data.id} />
<CommentForm entryId={post.data.id} />

EmDashHead

Автоматическая вставка SEO-тегов, Open Graph.

import { EmDashHead } from "emdash/ui";
<Base>
  <head>
    <EmDashHead />
  </head>
  ...
</Base>

HTTP API (REST)

Прямые запросы к /_emdash/api/* — обычно обёрнуты в клиентские библиотеки, но можно использовать напрямую:

Collections

GET /_emdash/api/collections/pages/entries?limit=10
POST /_emdash/api/collections/pages/entries
  { "data": { "title": "...", "content": [...] } }

Single entry

GET /_emdash/api/collections/pages/entries/:id
PATCH /_emdash/api/collections/pages/entries/:id
DELETE /_emdash/api/collections/pages/entries/:id

Media

POST /_emdash/api/media/upload  (multipart/form-data)
GET /_emdash/api/media/file/:id
DELETE /_emdash/api/media/file/:id

Authentication

POST /_emdash/api/auth/login
  { "email": "...", "password": "..." }
→ returns { token, user }

Error handling

Стандартные ошибки EmDash

Код Описание Пример
400 Bad Request (неверные поля) { error: "Validation failed", details: {...} }
401 Unauthorized { error: "Invalid credentials" }
403 Forbidden (недостаточно прав) { error: "Cannot delete" }
404 Not found { error: "Entry not found" }
409 Conflict (duplicate slug) { error: "Slug already exists" }
500 Internal error { error: "Database error", code: "SQLITE_BUSY" }

Pagination

Cursor-based пагинация (рекомендуется):

let cursor: string | undefined;
while (true) {
  const { entries, nextCursor } = await getEmDashCollection("posts", {
    limit: 20,
    cursor,
    orderBy: { published_at: "desc" },
  });
  // render entries
  if (!nextCursor) break;
  cursor = nextCursor;
}

Caching

Cache hints

Каждый запрос возвращает cacheHint:

const { cacheHint } = await getEmDashEntry("pages", "home");
Astro.cache.set(cacheHint);
// { ttl: 300, tags: ["collection:pages", "entry:home"] }

TTL (time-to-live):

  • pages: 600 сек (10 мин)
  • posts: 300 сек (5 мин)
  • media: 86400 сек (1 день)

Rate limiting

По умолчанию:

  • API: 100 запросов/мин на IP
  • Админка: 200 запросов/мин

Можно настроить в astro.config.mjs.
