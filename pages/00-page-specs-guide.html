Дашборд/Страницы/Page specs guide

Правила написания page-specs

Файлы в docs/page-specs/ — это ТЗ по каждой странице. Они нужны, чтобы:

  • Агент или разработчик понимал структуру страницы до начала кода
  • Было легко вернуться к странице позже и вспомнить решения
  • Все spec'и были единообразны

Именование

Файл называется по slug страницы: index.md, kontakty.md, o-shkole.md, czeny.md, 0klass.md.

Обязательные разделы

Каждый spec обязательно содержит:

1. Заголовок и статус

# Название страницы (/slug)

## Статус: ⬜ Не начата / 🔨 В работе / ✅ Готова

2. Контент

Откуда берутся данные. Для каждого блока на странице:

| Данные | Откуда |
|--------|--------|
| Заголовок | `pages` collection, slug `o-shkole` |
| Список X | `teachers` collection, orderBy `data.order` asc |
| Текст Y | Статический (из rksh-msk.ru) |

Если используется коллекция — перечислить используемые поля:

### Поля коллекции teachers

| Поле | Тип | Используется как |
|------|-----|-----------------|
| `full_name` | string | Заголовок карточки |
| `photo` | image | Аватар (fallback: инициалы) |

3. Структура страницы

Визуальная схема — ASCII-арт или нумерованный список секций с указанием:

  • Понятного названия блока
  • Источника данных (CMS / статика)
  • Фона секции (если отличается от --color-bg)
  • Ключевых визуальных элементов
## Структура страницы

1. **Hero** — Фон: --color-primary. h1 + announce + CTA. Данные: статика
2. **Преимущества** — Фон: --color-bg. 6 карточек из CMS (advantages)
3. **CTA-баннер** — Фон: --color-primary. Кнопка: --color-accent-warm

Условные разделы

Включать только если применимо. Если раздел пуст — не включать.

4. Отличия от гайда

Только отклонения от 01-design-guide.md. Если всё по гайду — раздел не нужен.

| Элемент | По гайду | Здесь | Причина |
|---------|----------|-------|---------|
| h1 размер | --font-size-4xl | --font-size-5xl | Крупный акцент на главной |

5. Компоненты

Какие файлы участвуют. Включать если есть специфичные компоненты или если страница сложная.

| Компонент | Файл | Тип |
|-----------|------|-----|
| Base layout | `src/layouts/Base.astro` | Общий |
| Шаблон | `src/pages/o-shkole.astro` | Специфичный |
| ScheduleTable | `src/components/ScheduleTable.astro` | Общий |

6. Отличия от оригинала

Сравнение с rksh-msk.ru. Включать для первых страниц или если есть существенные отличия.

7. SVG-иконки

Если на странице есть иконки — перечислить их с описанием path и цветом.

8. TODO

Что нужно доделать. Только конкретные задачи, не абстрактные.

Порядок работы

  1. До кода — создать spec со статусом «⬜ Не начата», заполнить Контент и Структуру
  2. Во время — обновлять статус на «🔨 В работе», добавлять отличия от гайда
  3. После — статус «✅ Готова», заполнить все разделы, обновить 00-pages-overview.md

Чего не писать

  • Не дублировать дизайн-гайд — только отличия
  • Не писать CSS-код (кроме ключевых решений в «Отличиях»)
  • Не описывать Base layout, Header, Footer — они общие для всех страниц
  • Не придумывать контент — берём только с rksh-msk.ru или из CMS
