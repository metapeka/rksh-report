Решение проблем: troubleshooting

Ошибки запуска dev-сервера

createPlugin is not a function

Симптомы:

[ERROR] TypeError: (0 , __vite_ssr_import_0__.createPlugin) is not a function
    at eval (virtual:emdash\plugins:9:16)

Причина: Плагин экспортирует export default definePlugin(...), а EmDash ожидает named export createPlugin.

Решение:

// plugins/rksh-blocks/src/sandbox-entry.ts
// ❌ НЕПРАВИЛЬНО:
export default definePlugin({...});

// ✅ ПРАВИЛЬНО:
export function createPlugin(options = {}) {
  return definePlugin({
    id: "rksh-blocks",
    version: "1.0.0",
    admin: { ... },
  });
}

Зачем: EmDash загружает плагин через import { createPlugin } from "plugin-entrypoint".

Content config not loaded

Симтом: 

[WARN] [content] Content config not loaded

Причина: Не найден seed/seed.json или ошибка синтаксиса JSON.

Решение:

  1. Проверить, что файл существует: ls seed/seed.json
  2. Валидировать JSON:
npx emdash seed seed/seed.json --validate
  1. Если ошибка — исправить (пропущена запятая, невалидный тип поля)

Address already in use (порт 4321 занят)

Причина: другой процесс уже слушает порт.

Решение:

# Найти PID
netstat -ano | findstr :4321
# или
lsof -i :4321

# Убить процесс
taskkill /PID <PID> /F  # Windows
kill -9 <PID>          # Linux/Mac

# Или изменить порт в .env
PORT=4322 npm run dev

Database locked / SQLite busy

Причина: несколько processes пишут в SQLite одновременно (редко в dev).

Решение:

  1. Убедиться, что нет других инстансов dev-сервера
  2. В astro.config.mjs:
database: sqlite({
  url: "file:./data.db?cache=shared&_journal_mode=WAL",
}),

WAL mode позволяет concurrent reads/writes.

Ошибки в админке

Не могу войти (неверный пароль)

Решение:

  1. Проверить .env — нет ли EMDASH_SECRET?
  2. Сбросить пароль через CLI:
npx emdash user:reset-password --email=admin@rksh-msk.ru
  1. Если не помогает — удалить data.db и пересоздать (теряется весь контент!)

Странности в админке (UI глючит)

Причина: кэш Vite устарел.

Решение:

# Очистить кэш
rm -rf .astro/
rm -rf node_modules/.vite

# Перезапустить
npm run dev

Меню не обновляется после редактирования

Причина: кэш контента не инвалидировался.

Решение:

  1. Убедиться, что на странице есть Astro.cache.set(cacheHint)
  2. В dev-режиме кэш disabled, в prod — проверить TTL
  3. Вручную очистить кэш (если Redis):
redis-cli FLUSHALL

Проблемы с контентом

Изображения не отображаются

Проверка:

  1. В админке: Медиа → найти файл → проверить URL (должен быть /_emdash/api/media/file/...)
  2. На фронте: <img src={image.src}> — network tab в DevTools — 404?
  3. Папка uploads/ существует? Права на чтение?

Решение:

# Проверить, что файл на диске
ls uploads/2025/04/15/photo.jpg

# Если нет — перезагрузить через админку

Если изображение в Portable Text:

import { Image } from "emdash/ui";
<Image image={entry.data.image} alt="..." />

Нельзя: <img src={entry.data.image} /> — будет [object Object].

Блоки не появляются в / меню редактора

Причины:

  1. Плагин не загружен — серверные логи: createPlugin is not a function
  2. Dev-сервер не перезапускался после изменений в плагине
  3. В astro.config.mjs нет plugins: [rkshBlocks()]

Решение:

  1. Перезапустить dev completely (не просто HMR)
  2. Проверить astro.config.mjs:
import { rkshBlocks } from "@rksh/blocks";
emdash({
  plugins: [rkshBlocks()],
}),
  1. Проверить, что @rksh/blocks установлен: ls node_modules/@rksh/blocks

Portable Text отображает [object Object]

Причина: в контенте image объект, а вы выводите как строку.

Решение:

<!-- ❌ -->
<img src={post.data.image.src} />

<!-- ✅ (правильный компонент EmDash) -->
<Image image={post.data.image} alt={post.data.image.alt} />

Slug не сохраняется / дублируется

Причина: slug должен быть уникальным в коллекции.

Решение:

  1. В админке: редактировать slug вручную
  2. Если «already exists» — добавить цифру (o-shkole-2)
  3. Влиять на URL — менять slug, но тогда старый URL сломается. Нужен redirect.

404 на странице после публикации

Проверка:

  1. Slug correct? /o-shkole/ в меню
  2. Файл src/pages/pages/[slug].astro существует
  3. Запись в БД: sqlite3 data.db "SELECT * FROM entries WHERE collection='pages' AND slug='o-shkole'"
  4. Статус published?

Решение: если запись есть, но 404 — возможно, роут не зарегистрировался. Проверить getStaticPaths().

Ошибки TypeScript

TS2307: Cannot find module

EmDash типы не сгенерированы.

Решение:

npx emdash types
# или
npm run typecheck  #Astro check

Если ошибка persists — удалить tsconfig.tsbuildinfo и emdash-env.d.ts, перезапустить dev.

Type mismatch в entry.data

Пример: Property 'excerpt' does not exist on type...

Причина: emdash-env.d.ts устарел (коллекция изменилась, типы не регенерировались).

Решение:

npx emdash types --force
# или удалить файл и перезапустить dev
rm emdash-env.d.ts
npm run dev

Производительность

Страницы медленно грузят

Диагностика:

  1. Открыть DevTools → Network → проверить время ответа
  2. Посмотреть server logs: запрос к БД длится > 200 мс?

Оптимизации:

  1. Кэширование: убедиться, что Astro.cache.set(cacheHint) есть на всех страницах
  2. Индексы БД: sqlite3 → CREATE INDEX idx_entries_collection_slug ON entries(collection, slug);
  3. N+1 запросы: не делать запрос в цикле. Использовать include или batch.
  4. Оптимизировать изображения: сжать, WebP, lazy load.

Высокое CPU / Memory

Причины:

  • Большое количество simultaneous connections (нет connection pooling в SQLite)
  • Утечка памяти в плагине (closure, global variable)

Решение:

  1. top / htop — посмотретьProcess
  2. Если Node > 1 GB — investigate heap dump
  3. Ограничить параллельные запросы (через Nginx limits)

Безопасность

Подозрительные файлы в uploads/

.php, .js, .exe — потенциальные backdoors.

Решение:

# Найти dangerous extensions
find uploads/ -type f \( -name "*.php" -o -name "*.js" -o -name "*.exe" \)
# Удалить
find ... -delete
# Заблокировать в storage config (allowlist)

SQL Injection

EmDash использует параметризованные запросы (Kysely), инъекций нет.
Но если пишете raw queries — используйте параметры:

db.raw("SELECT * FROM entries WHERE slug = ?", [slug]);

Never string concatenation.

Сеть

Cannot connect to localhost:4321

Причины:

  • Сервер не запущен
  • Порт другой (смотрите консоль при старте)
  • Фаервол блокирует

Решение:

# Проверить, слушает ли порт
netstat -an | grep 4321

# Запустить сервер
npm run dev

# Проверить логи
cat .astro/console.log  # или смотреть в окне dev.bat

Медиа

Превышен лимит размера файла

Ошибка: File too large (max 10MB)

Решение:

  1. Увеличить лимит в storage config:
storage: local({
  directory: "./uploads",
  maxFileSize: 50 * 1024 * 1024, // 50 MB
}),
  1. Проверить Nginx/proxy limits: client_max_body_size

Broken images after migration

Причина: URL в контенте старые (/wp-content/...), а файлы лежат в uploads/ с другими именами.

Решение:

  1. Обновить все записи:
UPDATE entries
SET content = replace(content, 'https://rksh-msk.ru/wp-content/', '/_emdash/api/media/file/')
WHERE collection='pages';
  1. Или скриптом на Node пройтись по всем entries и обновить image.src

Частые вопросы (FAQ)

«Как очистить кэш?»

# EmDash кэш (in-memory) — перезапуск сервера
#Redis кэш
redis-cli FLUSHALL
#Vite кэш
rm -rf node_modules/.vite

«Как увидеть SQL-запросы?»

Включить логирование:

// src/middleware.ts
import { getDb } from "emdash/db";
getDb().on("query", (query) => console.log("[SQL]", query));

«Как увеличить лимит загрузки?»

В astro.config.mjs:

emdash({
  storage: local({
    maxFileSize: 50 * 1024 * 1024, // 50 MB
  }),
}),

«Нет места на диске»

# Проверить
df -h
# Очистить логи
rm -rf logs/*
# Очистить неиспользуемые Docker образы
docker system prune -a
# Удалить старые бэкапы
find /backups -type f -mtime +30 -delete

«Почему в продакшне 500 error?»

  1. Проверить логи: pm2 logs rksh-web или docker logs <container>
  2. Убедиться, что .env.production загружен (переменные окружения)
  3. Проверить, что папка uploads/ доступна на запись
  4. Проверить подключение к БД (файл data.db существует)

«Как посмотреть ошибки в production?»

Если Docker:

docker logs -f <container_name>

Если PM2:

pm2 logs rksh-web
# или
tail -f ~/.pm2/logs/rksh-web-error.log

Исправление багов

Шаги:

  1. Воспроизвести: чётко описать шаги, которые приводят к ошибке
  2. Собрать логи: серверные, браузерные console, network tab

Что делать, если ничего не помогает

  1. Очистить кэш полностью:
rm -rf .astro/
rm -rf node_modules/.vite
rm -rf node_modules/.cache
npm run dev
  1. Переустановить зависимости:
rm -rf node_modules package-lock.json
npm install
  1. Удалить БД и пересоздать (данные потеряются!):
rm data.db
rm -rf uploads/
npx emdash dev  # создаст пустую БД из seed
  1. Клонировать заново и переустановить

Профилактика

  • Не редактировать data.db вручную (кроме SELECT)
  • Не коммитить data.db в git (в .gitignore)
  • Регулярно бэкапить (uploads/, data.db)
  • Не удалять поля из seed.json в production без миграции
  • Писать миграции при изменении schema

Checklist для быстрой диагностики

Сервер не запускается:

  • npm install прошёл без ошибок
  • EMDASH_SECRET установлен
  • Порт свободен (4321)
  • data.db не заблокирован
  • Логи: npm run dev 2>&1 | tee log.txt

Страница 404:

  • Slug совпадает с URL
  • Статус published
  • getEmDashEntry возвращает entry (console.log)
  • Роут [slug].astro существует

Изображение не грузится:

  • Network tab → статус 200?
  • URL правильный (/_emdash/api/media/file/...)
  • Файл на диске есть
  • alt текст не пустой (не влияет, но для валидности)

Форма не отправляется:

  • JS console → ошибки?
  • Network → POST request → статус 200?
  • CORS? (если фронт и бэкенд на разных доменах)
  • honeypot заполнен? (спам-фильтр)
