SEO и мета-теги

Автоматическая генерация SEO

EmDash автоматически генерирует базовые мета-теги для каждой страницы на основе:

  • Поля title (заголовок страницы)
  • Поля excerpt или первых 160 символов content
  • Настроек сайта (settings.site_name)
  • Изображения og:image (если указано)

Базовые мета-теги:

<title>{title} | {site_name}</title>
<meta name="description" content="{excerpt}" />
<meta property="og:title" content="{title}" />
<meta property="og:description" content="{excerpt}" />
<meta property="og:type" content="website" />
<meta property="og:url" content="{canonical_url}" />
<meta name="twitter:card" content="summary_large_image" />

Настройка SEO на уровне сайта

Админка: Настройки сайта → SEO

Настройка SEO на уровне страницы

В админке (если включено)

При редактировании страницы/новости есть вкладка SEO:

Поле Описание Приоритет
meta_title Заменяет {title} в <title> Высокий
meta_description Кастомное описание (иначе excerpt) Высокий
meta_keywords Устарело, но можно добавить Низкий
og_image Изображение для соцсетей (переопределяет default) Средний
noindex Не индексировать страницу (для черновиков) Высокий
canonical_url Канонический URL (при дублях) Высокий

Структура заголовков (H1-H6)

Правило: одна H1 на страницу.

На страницах, созданных через pages/[slug].astro:

  • H1 — это page.data.title (отображается в шаблоне)
  • Подзаголовки — внутри content (Portable Text)

Пример:

<h1>О школе</h1> <!-- из title -->
<p>...контент...</p>
<h2>Наши принципы</h2> <!-- из Portable Text -->
<h3>Принцип 1</h3>

URL (слаг) и permalinks

Формат slug: латиница, дефисы, lowercase.

Примеры:

  • o-shkole (О школе)
  • prepodavateli (Преподаватели)
  • nachalnaya-shkola (Начальная школа)

Рекомендации:

  • Краткие, запоминаемые
  • Без стоп-слов (the, and, для)
  • Ключевые слова: czeny (цены), otzyvy (отзывы)

Изменение slug: можно в админке, но будет 301 редирект? Нет — EmDash не создаёт автоматически. Нужно:

  1. Вручную добавить redirect в src/middleware.ts
  2. Или использовать пакет astro-redirects

Sitemap (карта сайта)

Автогенерация: EmDash создаёт sitemap по маршрутам.

Доступ: /_emdash/api/sitemap.xml (или /sitemap.xml?)

Что включено:

  • Все опубликованные pages
  • Все posts
  • Коллекции (если настроены)

Настройка:

// src/pages/sitemap.xml.ts или через middleware
import { getSitemap } from "emdash";
const sitemap = await getSitemap();
return new Response(sitemap, { headers: { "Content-Type": "application/xml" } });

РеFrequency:

  • Страницы: weekly
  • Новости: daily (часто обновляются)

robots.txt

Создать файл: src/pages/robots.txt

User-agent: *
Allow: /

Sitemap: https://rksh-msk.ru/sitemap.xml

Disallow для админки:

User-agent: *
Disallow: /_emdash/
Disallow: /api/

Open Graph (соцсети)

Автоматически: EmDash добавляет базовые OG-теги.

Кастомизация:

---
import { getSeoMeta } from "emdash";
const seo = await getSeoMeta({
  title: page.data.title,
  description: page.data.excerpt,
  image: page.data.og_image?.src || defaultOgImage,
  url: Astro.url,
  type: "article" // для новостей
});
---
<head>
  {seo.tags.map(tag => <Astro.self {...tag} />)}
</head>

Canonical URL

Зачем: избегать дублей контента.

Как: в <head>:

<link rel="canonical" href="https://rksh-msk.ru/o-shkole/" />

Генерировать автоматически:

const canonical = Astro.url.pathname;
// Если есть query params? игнорировать

SEO: noindex, follow в meta robots.

noindex, nofollow

Когда использовать:

  • Черновики (status=draft) — автоматически
  • Administrator-only страницы (админка)
  • Простые страницы («Спасибо за заявку»)

Как:

<meta name="robots" content="noindex, nofollow" />

Локализация (i18n)

Пока: только русский. Если понадобится английский:

  1. Добавить поле lang в коллекции (ru, en)
  2. Создать src/pages/en/[slug].astro
  3. Динамически подставлять lang в html tag
  4. hreflang link tags:
<link rel="alternate" hreflang="ru" href="https://rksh-msk.ru/o-shkole/" />
<link rel="alternate" hreflang="en" href="https://rksh-msk.ru/en/about/" />

Производительность SEO

Core Web Vitals

  • LCP (Largest Contentful Paint) < 2.5s

    • Оптимизировать изображения (WebP, lazy load)
    • Критичный CSS inline
    • Fonts preload

  • FID (First Input Delay) < 100 ms

    • Минимизировать JS
    • Defer non-critical

  • CLS (Cumulative Layout Shift) < 0.1

    • Задать width/height для изображений
    • Reserve space for ads/embeds

Мониторинг SEO

  1. Google Search Console — индексация, ошибки, impressions
  2. Yandex.Webmaster — аналогично
  3. Screaming Frog — аудит мета-тегов
  4. Ahrefs / SEMrush — позиции по ключам

Чек-лист SEO

  • У всех страниц уникальный title (50-60 символов)
  • Уникальный description (150-160 символов)
  • Нет noindex на основных страницах
  • Канонические URL указаны
  • OG-теги заполнены (title, description, image, url)
  • Структурированные данные (Schema.org) есть на главной
  • Карта сайта (sitemap.xml) доступна и отправлена в поисковики
  • robots.txt не блокирует важные страницы
  • URL читаемые (без параметров ?id=123)
  • H1 одна на страницу, не дублируется
  • Изображения имеют alt
  • Скорость загрузки: LCP < 2.5s (PageSpeed Insights)

Миграция SEO со старого сайта

  1. Спарсить все URL со старого rksh-msk.ru (38 страниц)
  2. Сопоставить slugs:
    • /o-shkole/ → /o-shkole/ (точно)
    • /czeny/ → /czeny/
    • ...
  3. Создать 301 redirects:
// src/middleware.ts
if (oldUrl.pathname === "/old-about") {
  return Response.redirect(new URL("/o-shkole/", oldUrl), 301);
}
  1. Обновить sitemap — указать новые URL
  2. Проверить в Search Console — нет 404 после миграции
