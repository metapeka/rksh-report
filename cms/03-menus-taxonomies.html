Таксономии (теги, категории)

EmDash поддерживает таксономии (taxonomies) — классификаторы контента. В текущем проекте не используются, но могут понадобиться.

Предлагаемая таксономия для новостей

Имя: news_category
Тип: hierarchical (иерархические категории)

Новости (news)
└─ Собрания (meetings)

Как используется:

  • Каждой новости можно присвоить категорию «Новости» или «Собрания»
  • На странице /news/ можно отфильтровать по категории
  • URL: /news/category/novosti/

Как добавить таксономию

  1. В seed/seed.json:
"taxonomies": [
  {
    "name": "news_category",
    "label": "Категории новостей",
    "hierarchical": true,
    "fields": [...]
  }
]
  1. Запустить npx emdash dev — создаст таблицуterms`
  2. В админке: Настройки сайта → Таксономии → добавить термины

Связь меню с коллекциями

Меню primary и secondary содержат ссылки (URL). Они не связаны напрямую с коллекциями EmDash, а просто ведут на маршруты:

  • custom — произвольный URL (например /, /news/)
  • collection — автоматическая ссылка на запись из коллекции (пока не поддерживается EmDash? Уточнить)
  • taxonomy — на архив таксономии

Рекомендация: использовать slugs как URL:

  • Страница «О школе» → slug=o-shkole → URL /o-shkole/
  • Коллекция teachers → список на /prepodavateli/ (но как указать? через custom)

SSR vs Static

Все страницы с меню — SSR. Меню запрашиваются через getMenu() на сервере:

import { getMenu } from "emdash";
const menu = await getMenu("primary"); // returns { items: [...] }

Меню кэшируются вместе со страницей (через cacheHint). При изменении меню в админке — инвалидация кэша.

Динамические меню (будущее)

Сейчас меню управляются вручную. Можно автоматизировать:

  • Автоматическое второе меню — собирать из收藏 (collection) «Программы»
  • Меню "Уровни школы" — динамически генерировать из school_levels
  • Бreadcrumbs — на основе parent_page связи

Для этого нужно:

  1. Создать коллекцию menu_items (или использовать существующую pages с parent)
  2. В SecondMenu.astro заменить getMenu("secondary") на запрос коллекции
  3. Добавить drag-and-drop сортировку в админке

Практические задачи

Задача 1: Добавить пункт «Сведения» в основное меню

  1. Админка → Настройки сайта → Меню → primary
  2. Нажать «+ Добавить пункт»
  3. Label: Сведения об образовательной организации
  4. URL: /sveden/
  5. Сохранить
  6. Проверить на фронте — появился пункт меню

Задача 2: Создать выпадающий «Курсы для дошкольников»

  1. В меню primary (или secondary) создать родительский пункт:
    • Label: «Курсы для дошкольников»
    • Тип: custom, URL: /doshkolniki/
    • Отметить «Has children»
  2. Добавить дочерние пункты:
    • «Подготовка к школе от 4 до 7 лет» → /doshkolniki/podgotovka-k-shkole-ot-4-do-7-let/
    • ...
  3. Сохранить → на фронте должно появиться dropdown

Задача 3: Сортировка меню

В админке: drag & drop. Порядок в seed.json — это порядок отображения.

SEO и доступность

  • Меню должны быть семантичными: <nav aria-label="Основное меню">
  • Dropdown-кнопки: aria-expanded, aria-controls — уже в Header.astro
  • Фокус-статы: есть в global.css (:focus-visible)
  • На мобильных: hamburger menu (пока не реализован, есть только кнопка)

Частые проблемы

Проблема Причина Решение
Пункт меню не кликабелен href="#" или пустой URL Указать корректный slug
Dropdown не открывается JS error в Base.astro Проверить консоль, ARIA attrs
Меню не обновляется после изменений Кэш не инвалидировался Перезапустить dev или очистить кэш
Второе меню не отображается secondary пусто в БД Заполнить через админку
