Дашборд/CMS/Content workflow

Workflow: создание и редактирование контента

Доступ к админке

URL: http://localhost:4321/_emdash/admin
Первоначальная настройка: при первом запуске npx emdash dev создаётся админ-пользователь. Логин/пароль можно посмотреть в консоли или в .env.

Общий процесс создания страницы

1. Нажать «Контент» → выбранная коллекция

Для статической страницы: Контент → Страницы
Для новости: Контент → Новости
Для преподавателя: Контент → Преподаватели (если создана коллекция)

2. «Создать запись»

Откроется форма с полями, определёнными в схеме коллекции (настраивается через Admin UI).

3. Заполнить основные поля

  • Заголовок — отображается в админке и используется в <title> страницы
  • Slug — генерируется автоматически из заголовка (транслитерация). Можно редактировать вручную.
    Правила: латиница, цифры, дефисы, без пробелов. Уникальный в пределах коллекции.
  • Статус:
    • Черновик — не виден на сайте
    • Опубликован — доступен публично
    • В архиве — скрыт, но сохранён
  • Дата публикации — если требуется отложенная публикация

4. Редактор Portable Text

Основное поле content (тип portableText). Интерфейс:

[Текст] Bold Italic Heading Link
[Блоки] /image /video /table ...

Добавить блок:

  1. Поставить курсор в нужное место
  2. Нажать / (слеш) → открывается меню доступных блоков
  3. Выбрать блок → открывается форма Block Kit с полями
  4. Заполнить поля → нажать «Сохранить» (или Enter)
  5. Блок вставляется в поток

Редактировать блок:

  • Кликнуть на блок → панель с полями становится редактируемой
  • Изменить значения → автосохранение или Ctrl+S

Удалить блок:

  • Выделить блок → Backspace / Delete

Переместить блок:

  • Drag handle (6 точек) слева от блока → перетащить

5. SEO-поля (если включены)

В разделе «SEO» (вкладка):

  • Meta title — если пусто, используется title
  • Meta description — краткое описание для поисковиков
  • Open Graph image — изображение для соцсетей

6. Сохранить и опубликовать

  • Сохранить (Ctrl+S) — сохраняет черновик
  • Опубликовать — меняет статус на published и делает доступным на сайте

После публикации:

  • Кэш инвалидируется (через cacheHint)
  • Страница становится доступна по URL

Работа с кастомными блоками (плагин @rksh/blocks)

Как добавить блок на страницу

  1. В редакторе content нажать /
  2. В списке появится новый раздел «Кастомные блоки» (или просто в общем списке):
    • Hero-секция
    • Особенности
    • Принципы обучения
    • Преимущества
    • Расписание
    • Преподаватели
    • CTA-секция
  3. Выбрать блок → заполнить форму
  4. Нажать «Сохранить» → блок появился в контенте

Пример: добавить Hero на главную

  1. Контент → Страницы → Создать запись (или найти «Главная»)
  2. Заголовок: Главная (slug должен быть home или index — нужно проверить роутинг)
  3. В content нажать / → выбрать Hero-секция
  4. Заполнить:
    • Заголовок: Частная школа РКШ — Москва, Куркино
    • Подзаголовок: Идёт набор учеников на 2025–2026 учебный год
    • Текст кнопки: Получить презентацию
    • Ссылка кнопки: /kontakty (или #presentation)
  5. Нажать / ещё раз → Особенности → заполнить 6 карточек
  6. Нажать / → Преимущества → заполнить 6 преимуществ
  7. Нажать / → CTA-секция → «Остались вопросы?» → /kontakty
  8. Сохранить → Опубликовать

Результат: главная страница состоит из 4 блоков.

Версионирование и история

EmDash поддерживает revisions (если включено в коллекции). Для pages и posts включено.

Как посмотреть историю:

  1. Открыть запись
  2. Нажать кнопку «История» (или «Ревизии»)
  3. Выбрать версию → «Восстановить»

Каждое сохранение создаёт новую revision. Можно сравнить изменения.

Публикация и расписание

  • Немедленная публикация: статус published → контент сразу доступен
  • Отложенная публикация: задать publish_date → страница появится в указанное время
  • Снятие с публикации: статус archived → страница возвращает 404

Поиск и фильтры в админке

В списках записей:

  • Поиск по заголовку и содержимому (если поле searchable: true)
  • Фильтр по статусу (draft/published/archived)
  • Фильтр по тегам/категориям (если есть таксономии)

Bulk-операции

  • Массовое изменение статуса — выделить несколько записей → «Изменить статус»
  • Удаление — выделить → «Удалить» (переходит в корзину)
  • Экспорт — через API (/_emdash/api/collections/.../entries)

Медиа-библиотека

Загрузка изображений:

  1. Админка → Медиа → Загрузить
  2. Выбрать файл (JPG, PNG, WebP, GIF)
  3. Заполнить Alt текст (для доступности)
  4. Сохранить

Использование в Portable Text:

  • В редакторе нажать /image
  • Выбрать из библиотеки или загрузить новое
  • Автоматически вставляется блок <Image />

URL-ы:

  • Dev: http://localhost:4321/_emdash/api/media/file/<id>
  • Prod: /media/<filename> (если настроен CDN)

Безопасность

  • Авторизация: логин/пароль (или OAuth, если настроено)
  • Роли: Admin, Editor, Author (настраивается в «Пользователи»)
  • Сессии: файловая система в dev, в prod — Redis (опционально)
  • Секретный ключ: EMDASH_SECRET в .env — должен быть надёжным

Частые ошибки

«Slug уже существует»

У двух страниц одинаковый slug. Изменить один из них.

«Не могу сохранить — поле обязательно»

Заполнить все поля с required: true (кроются под «asterisk» в админке).

«Изображение не отображается на сайте»

  • Проверить, что src начинается с /_emdash/api/media/file/
  • В src должен быть объект {src, alt} (не строка)
  • Использовать <Image image={post.data.image} />, а не <img src={...}>

«Контент не обновляется после редактирования»

  • Убедиться, что на странице есть Astro.cache.set(cacheHint)
  • В dev-режиме кэш disabled, в prod — проверить TTL

Грамотное использование блоков

Как не надо:

❌ Жёстко вшивать контент в шаблон:

<h1>Частная школа РКШ</h1>
<p>Мы такие же родители...</p>

Это нередактируемо.

✅ Использовать PortableText:

<Base title={page.data.title}>
  <PortableText value={page.data.content} />
</Base>

Контент полностью управляется из админки.

Как структурировать страницу «О школе»

Контент в админке (порядок блоков):

  1. [hero] — «О школе»
  2. [features] — 6 особенностей программы
  3. [principles] — 6 принципов обучения
  4. [teachers] — 3 ключевых педагога (выбор из коллекции, если будет)
  5. [advantages] — «Чем мы лучше обычной школы»
  6. [cta] — «Записаться на экскурсию»

Каждый блок — отдельная форма. Администратор может перетаскивать, удалять, дублировать.

Bulk-импорт контента

Если нужно загрузить много страниц сразу:

  1. Через API:
POST /_emdash/api/collections/pages/entries
{
  "data": {
    "title": "О школе",
    "slug": "o-shkole",
    "content": [{ "_type": "block", ... }]
  }
}
  1. Скриптом ( Node.js ):
import { createClient } from "emdash";
const client = createClient({ url, token });
await client.entries.create("pages", { ... });
  1. CSV → JSON → import (пока вручную, нет встроенного импорта CSV)

Проверка контента перед публикацией

Чек-лист:

  • Все заголовки без ошибок
  • Слаг корректный (латиница, дефисы)
  • Все обязательные поля заполнены
  • Изображения имеют alt текст
  • Нет «ломатых» ссылок (внутренние — слэш в начале)
  • SEO-поля заполнены (title, description)
  • Кнопки CTA ведут на правильные URL

Документирование структуры страниц

Важно: вести docs/cms/04-pages-templates.md (этот файл) в актуальном состоянии. Для каждой новой страницы:

  1. Определить, из какой коллекции
  2. Описать, какие блоки используются
  3. Зафиксировать slug

Пример:

Страница: О школе (/o-shkole/)
Коллекция: pages
Блоки: [hero, features, principles, teachers, advantages, cta]
Slug: o-shkole
