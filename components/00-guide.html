Как пользоваться каталогом компонентов

Структура spec'а компонента

Каждый файл 01-card.md, 02-section-title.md содержит разделы:

1. Заголовок + уровень + ссылки

# 01 · Card · Базовый

Универсальный контейнер с фоном, скруглением и бордером/тенью.

→ Дизайн-гайд: ../page-specs/01-design-guide.md
→ Код: src/components/ui/Card.astro
→ Showcase: /styleguide

Обязательные ссылки: дизайн-гайд, .astro-файл, styleguide.

2. Варианты

Таблица вариантов. Каждый вариант = значение пропа variant.

| Вариант | Когда | Фон | Бордер | Тень | Дефолт |
|---------|-------|-----|--------|------|--------|
| surface | По умолчанию | --color-surface | --color-primary-line | --shadow-sm | ✓ |
| elevated | Акцент | --color-surface | none | --shadow-lg | |
| muted | Приглушённый | --color-surface-alt | none | none | |

Столбец «Дефолт» — у какого варианта галочка ✓.

3. Использование (Astro)

Код-пример вызова компонента. Агент копирует отсюда.

<Card>Содержимое</Card>
<Card variant="elevated">Важное</Card>
<Card variant="muted">Мест нет</Card>

4. CSS-справка

Явно помечено как справка. Единая точка правды — .astro-файл.

Перечисление классов и ключевых токенов. Не полный CSS — только корневые классы и отличия.

5. Состав (для составных)

Из каких базовых компонент собран. Ссылки на их spec'и.

ProgramCard = Card variant=surface + Badge ×2 + Link

Базовые компоненты этот раздел пропускают.

6. Где используется

Список страниц с указанием варианта. Агент проверяет перед изменением — ломает ли другим.

| Страница | Контекст | Вариант |
|----------|----------|---------|
| Главная (/) | ProgramCard | surface |
| Цены (/czeny) | Тарифные карточки | elevated |

7. Связанные

Ссылки на другие компоненты из каталога (по номеру).

- [02 · SectionTitle](02-section-title.md)
- [03 · Badge](03-badge.md)

8. Адаптив (если отличается от гайда)

Только если компонент ведёт себя не по 01-design-guide.md.

Когда создавать новый компонент

  1. Паттерн повторился на 2+ страницах → вынести в базовый/составной
  2. Компонент сложный (больше 3 вложенных элементов) → описать даже если на 1 странице
  3. Компонент простой и на 1 странице → не описывать, оставить в page-spec

Когда обновлять существующий

  1. Добавился новый вариант → добавить строку в таблицу вариантов
  2. Изменились CSS-классы → обновить справку в spec
  3. Компонент стал использоваться на новой странице → добавить в «Где используется»
  4. Редизайн → обновить .astro-файл + spec, затем проверить все страницы из «Где используется»

Чего не писать в spec

  • Полный CSS-код — он в .astro-файле, spec даёт только справку
  • Дизайн-гайд целиком — компонент ссылается на него, не дублирует
  • Base layout, Header, Footer — они общие, не компоненты каталога
  • Контент — берём только с rksh-msk.ru или из CMS
