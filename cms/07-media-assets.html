Медиафайлы: загрузка, хранение, использование

Обзор

EmDash предоставляет встроенную систему медиа-хранения:

  • Локальное хранение: файлы сохраняются в папку uploads/
  • База данных: записи о файлах в коллекции media (автоматически)
  • Доступ: через API /_emdash/api/media/file/<id>
  • Админка: UI для загрузки (Медиа → Загрузить)

Хранение

Файловая система

project/
├── uploads/
│   ├── 2025/
│   │   ├── 04/
│   │   │   ├── 15/
│   │   │   │   ├── photo1.jpg
│   │   │   │   ├── photo2.png
│   │   │   │   └── doc1.pdf
│   │   │   └── ...
│   └── ...

Структура папок: uploads/{year}/{month}/{day}/{filename}
Зачем: организац

База данных

Таблица media (автоматически создаётся EmDash):

Поле Тип Описание
id ULID Уникальный ID
filename string Оригинальное имя файла
mime_type string image/jpeg, video/mp4, application/pdf
size number Размер в байтах
path string Путь на диске (относительно uploads/)
url string Публичный URL (/_emdash/api/media/file/<id>)
alt string Альтернативный текст (для изображений)
width number Ширина (для изображений)
height number Высота
uploaded_by string ID пользователя
created_at datetime Дата загрузки

Поддерживаемые форматы

Тип Расширения Ограничения
Изображения .jpg, .jpeg, .png, .gif, .webp, .svg Макс. размер: 10 МБ (можно настроить)
Видео .mp4, .webm, .mov Макс. 100 МБ, рекомендуется < 50 МБ
Документы .pdf, .doc, .docx Макс. 25 МБ
Аудио .mp3, .wav, .ogg Макс. 50 МБ

Важно:

  • SVG возможно содержит XSS — EmDash sanitizes?
  • sharp (опционально) для ресайза/конвертации изображений

Загрузка файлов

Через админку (ручная)

  1. /_emdash/admin → Медиа → Загрузить
  2. Выбрать файл (drag & drop или клик)
  3. Заполнить метаданные:
    • Название (title)
    • Alt-текст (для изображений, обязательно для доступности)
    • Описание (необязательно)
  4. Нажать «Сохранить»

Файл появится в библиотеке, получив id и url.

Через API (программно)

POST /_emdash/api/media/upload
Content-Type: multipart/form-data

file: <binary>

Ответ:

{
  "id": "01J...",
  "url": "/_emdash/api/media/file/01J...",
  "filename": "photo.jpg",
  "mime_type": "image/jpeg",
  "size": 245760,
  "width": 1920,
  "height": 1080
}

Использование в контенте

В Portable Text (редактор)

  1. В редакторе нажать / → выбрать «Image»
  2. Выбрать из библиотеки или загрузить новое
  3. Заполнить Alt и Caption
  4. Вставленный блок:
{
  "_type": "image",
  "asset": { "_id": "01J..." },
  "alt": "Логотип РКШ",
  "caption": "Основной логотип"
}

В Astro-компонентах

import { Image } from "emdash/ui";

const { entry } = await getEmDashEntry("pages", "home");
<Image image={entry.data.logo} alt={entry.data.logo.alt} />

Image компонент оптимизирует:

  • lazy loading (loading="lazy")
  • responsive srcset (если несколько размеров)
  • WebP fallback (если включён sharp)

Важно: image — это объект { src, alt, width, height }, не строка.

Оптимизация изображений

Автоматическая оптимизация (если установлен sharp)

EmDash может генерировать несколько размеров:

Original:  photo.jpg (4000x3000)
↓ resize
large:    photo.jpg?width=1200 (1200x900)
medium:   photo.jpg?width=800  (800x600)
small:    photo.jpg?width=400  (400x300)
thumb:    photo.jpg?width=200  (200x150)

Использование:

<Image
  image={entry.data.photo}
  widths={[400, 800, 1200]}
  sizes="(max-width: 768px) 100vw, 800px"
/>

Конвертация в WebP

Если браузер поддерживает:

<picture>
  <source srcset="photo.webp" type="image/webp" />
  <img src="photo.jpg" alt="..." />
</picture>

EmDash автоматически (если sharp установлен).

Видео

Загрузка

  1. Через админку (ограничение 100 МБ)
  2. Или через внешний хостинг (YouTube, Vimeo) — предпочтительнее

Рендеринг

Локальный видеофайл:

<video controls poster={thumb}>
  <source src={video.url} type="video/mp4" />
  Your browser does not support video.
</video>

YouTube/Vimeo:

<iframe
  src="https://www.youtube.com/embed/VIDEO_ID"
  title="..."
  allowfullscreen
></iframe>

Рекомендация: использовать YouTube/Vimeo для больших файлов. Локальные видео — только для коротких роликов (< 30 сек).

Документы (PDF, DOC)

Загрузка

Через админку или API.

Отображение

<a href={file.url} download class="doc-link">
  📄 {file.filename}
</a>

Или встроить PDF:

<embed src={file.url} type="application/pdf" width="100%" height="600px" />

CDN и production

Production-деплой

Файлы хранятся:

  1. Локально (если Docker volume)
  2. Или в облаке: S3, Yandex Cloud Object Storage

Настройка в astro.config.mjs:

storage: local({
  directory: "./uploads",
  baseUrl: "/_emdash/api/media/file", // dev
  // prod: "https://cdn.rksh-msk.ru/media"
})

Cloud storage (S3)

import { s3 } from "emdash/storage";
emdash({
  storage: s3({
    bucket: "rksh-media",
    region: "ru-central1",
    credentials: {
      accessKeyId: process.env.AWS_ACCESS_KEY,
      secretAccessKey: process.env.AWS_SECRET,
    },
  }),
});

Безопасность

Защита от вредоносных файлов

  1. Валидация MIME-типа (не только по расширению)
  2. Сканирование антивирусом (ClamAV, опционально)
  3. Размер файла: лимиты в astro.config.mjs или Nginx
  4. CORS: /_emdash/api/media/* — доступные для всех (публичные)

Доступ

  • Публичные файлы: /media/* — доступны без авторизации
  • Приватные: через signed URLs (если нужно) — опция в storage

Оптимизация производительности

Ленивая загрузка изображений

<Image
  image={photo}
  loading="lazy"
  decoding="async"
  fetchpriority="low"
/>

Предзагрузка kritischer Bilder

<link rel="preload" as="image" href="/media/logo.png" />

Кэширование на CDN

Headers:

Cache-Control: public, max-age=31536000, immutable

Для медиафайлов — год, потому они неизменяемы.

Миграция медиа со старого сайта

Сайт rksh-msk.ru использует WordPress. Нужно:

  1. Скачать все файлы из wp-content/uploads/
  2. Переименовать/реорганизовать под структуру uploads/{year}/{month}/{day}/
  3. Импортировать в EmDash:
    • Скрипт на Node.js читает папку
    • Для каждого файла создаёт запись в media collection через API
    • Сохраняет оригинальный путь в alt или custom field
  4. Обновить URL в контенте:
    • Было: https://rksh-msk.ru/wp-content/uploads/...
    • Стало: /_emdash/api/media/file/<id>
    • Или: /media/... (если CDN)

Удаление файлов

Через админку

  • Медиа → выбрать файл → Удалить
  • Удаляется запись из БД и файл с диска

Через API

DELETE /_emdash/api/media/file/<id>

Важно: проверять, не используется ли файл в других записях (каскадное удаление?).

Ограничения и проблемы

Проблема Причина Решение
Файл не загружается Превышен лимит размера Увеличить maxSize в конфиге storage
Изображение не отображается Нет sharp → нет ресайза Установить sharp: npm i sharp
Медленная загрузка больших файлов Синхронная запись на диск Использовать очередь (bullmq)
Конфликты имён Два файла с одинаковым именем EmDash добавляет timestamp?

Checklist для разработчика

  • Установлен sharp (для изображений) — npm i sharp --save-optional
  • Папка uploads/ существует и доступна для записи
  • В astro.config.mjs указан правильный storage: local(...)
  • Размеры файлов ограничены (10 МБ изображения, 100 МБ видео)
  • На production настроен CDN или Nginx кэширование
  • URL-ы в контенте используют /_emdash/api/media/file/...
  • Все изображения имеют alt текст (проверить через валидатор)
