Build и запуск

Development

# Установка зависимостей
npm install

# Запуск dev-сервера с hot reload
npm run dev
# или npx emdash dev
# или dev.bat (Windows)

# Admin UI: http://localhost:4321/_emdash/admin

Что делает npx emdash dev:

  1. Применяет миграции из seed/seed.json
  2. Создаёт/обновляет БД (SQLite)
  3. Генерирует TypeScript типы (emdash-env.d.ts)
  4. Запускает Astro dev server с Vite HMR
  5. Подгружает плагины (включая @rksh/blocks)

Production build

# Сборка
npm run build
# → creates ./dist/ with server entry

# Запуск
npm run start
# → node ./dist/server/entry.mjs

Checklist production:

  • NODE_ENV=production
  • EMDASH_SECRET установлен и надёжный
  • DATABASE_URL указывает на persistent storage
  • Папка uploads/ монтируется как volume (не удаляется при redeploy)
  • .env.production добавлен в .gitignore
  • Logging перенаправлен в файл или stdout (Docker logs)

Docker deployment (рекомендуется)

Dockerfile

# Stage 1: deps
FROM node:20-alpine AS deps
WORKDIR /app
COPY package.json package-lock.json ./
RUN npm ci --only=production

# Stage 2: builder
FROM node:20-alpine AS builder
WORKDIR /app
COPY --from=deps /app/node_modules ./node_modules
COPY . .
RUN npm run build

# Stage 3: runner ( slim )
FROM node:20-alpine AS runner
WORKDIR /app
ENV NODE_ENV=production

# Копируем зависимости и сборку
COPY --from=deps /app/node_modules ./node_modules
COPY --from=builder /app/dist ./dist
COPY --from=builder /app/data.db ./data/  # если БД локальная
COPY --from=builder /app/seed ./seed

# Создаём non-root пользователя
RUN addgroup --system --gid 1001 nodejs && \
    adduser --system --uid 1001 astro
USER astro

EXPOSE 4321
CMD ["node", "dist/server/entry.mjs"]

docker-compose.yml

version: "3.8"

services:
  web:
    build: .
    ports:
      - "4321:4321"

    volumes:
      - redis_data:/data

volumes:
  data:

  redis_data:

Миграции базы данных

Что такое миграции?

Изменения seed.json (новые поля, коллекции) применяются автоматически при запуске emdash dev или emdash migrate.

Команды

# Применить миграции (dev и prod)
npx emdash migrate

# Показать статус миграций
npx emdash migrate:status

# Откатить последнюю (осторожно!)
npx emdash migrate:rollback

Как безопасно менять schema

  1. Добавить поле — безопасно (старые записи получат null)
  2. Удалить поле — данные поля потеряются навсегда
  3. Изменить тип поля — может вызвать ошибки, нужен скрипт конвертации
  4. Переименовать коллекцию — лучше создать новую, скопировать данные

Best practice: Never delete fields in production. Mark as deprecated in seed, stop using in code, but keep DB column.

Backup strategy

Локальный backup (cron)

# /etc/cron.d/rksh-backup
0 2 * * * root /app/backup.sh

backup.sh:

#!/bin/bash
BACKUP_DIR=/backups
DATE=$(date +%Y%m%d_%H%M%S)
sqlite3 /app/data/data.db ".backup $BACKUP_DIR/data-$DATE.db"
tar -czf $BACKUP_DIR/uploads-$DATE.tar.gz /app/uploads
# Удалить старые (>30 дней)
find $BACKUP_DIR -name "data-*.db" -mtime +30 -delete

Cloud backup (Yandex Cloud Object Storage)

# Установить yc CLI
yc storage cp /backups/data-*.db storage://rksh-backups/

Сети и безопасность

Firewall (настройка сервера)

# Разрешить только 80/443
ufw allow 22    # SSH
ufw allow 80    # HTTP
ufw allow 443   # HTTPS
ufw deny 4321   # закрыть порт 4321 снаружи (будет behind nginx)

Nginx как reverse proxy

server {
    listen 80;
    server_name rksh-msk.ru;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name rksh-msk.ru;

    ssl_certificate /etc/letsencrypt/live/rksh-msk.ru/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/rksh-msk.ru/privkey.pem;

    location / {
        proxy_pass http://localhost:4321;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }

    # Статика (если нужно)
    location /_emdash/api/media/file/ {
        alias /app/uploads/;
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    # Запретить доступ к админке снаружи (опционально IP whitelist)
    location /_emdash/admin {
        allow 127.0.0.1;  # только локально
        deny all;
    }
}

SSR vs SSG

Проект использует SSR (output: "server") — потому что контент динамический из БД.

Если нужен SSG:

// astro.config.mjs
export default defineConfig({
  output: "static",  // гибридный режим (не подходит для CMS)
  // Или hybrid: "hybrid"
});

Но для EmDash только SSR.

Кэширование

CDN (Cloudflare / Yandex Cloud CDN)

Настройка:

  • CDN в front of Node.js app
  • Cache everything для статики (/media/*, /dist/*)
  • Bypass cache для /_emdash/* (админка, API)

Cache headers (в Nginx):

location / {
  proxy_pass http://node:4321;
  proxy_cache mycache;
  proxy_cache_valid 200 10m;
}

In-memory cache (EmDash)

EmDash кэширует результаты запросов к БД в памяти (или Redis). Управляется через cacheHint.

TTL по умолчанию: 5 минут (настраивается в коллекции settings).

Rate limiting

Защита API

// src/middleware.ts
import { rateLimit } from "astro/middleware";
export const onRequest = rateLimit({
  duration: 60_000,  // 1 минута
  limit: 100,        // 100 запросов
  namespace: "global",
});

Разные лимиты:

  • / — публичная, 1000/мин
  • /_emdash/api/* — 100/мин (для editors)
  • /api/forms/submit — 10/мин (защита от спама)

Скрипты_deploy

deploy.sh (bash)

#!/bin/bash
set -e

echo "=== Deploy RKSH ==="

# 1. Pull latest
git pull origin main

# 2. Install deps
npm ci --only=production

# 3. Build
npm run build

# 4. Apply migrations
npx emdash migrate

# 6. Clear CDN cache (Cloudflare)

  -H "Content-Type: application/json" \
  --data '{"purge_everything":true}'

  1. Купить домен (already: rksh-msk.ru)

Backups

Daily backup to Yandex Cloud

#!/bin/bash
# /app/backup.sh
DATE=$(date +%Y%m%d)
# DB backup
sqlite3 /app/data/data.db ".backup /backup/data-$DATE.db"
# Uploads backup
tar -czf /backup/uploads-$DATE.tar.gz /app/uploads
# Upload to Yandex Cloud
yc storage cp /backup/data-$DATE.db storage://rksh-backups/
yc storage cp /backup/uploads-$DATE.tar.gz storage://rksh-backups/
# Retention: keep last 30 days
find /backup -type f -mtime +30 -delete

Cron: 0 3 * * * /app/backup.sh

Recovery

  1. Остановить сервер: pm2 stop rksh-web
  2. Восстановить БД: sqlite3 data.db ".restore backup/data-20250415.db"
  3. Восстановить uploads: tar -xzf backup/uploads-20250415.tar.gz -C /app/
  4. Запустить: pm2 start rksh-web

Troubleshooting deployment

Ошибка: "Cannot find module 'sharp'"

npm install sharp --save-optional
# Или в Dockerfile: RUN npm ci --include=dev && npm rebuild sharp

Ошибка: "Address already in use"

Process уже использует порт 4321:

# Найти PID
lsof -i :4321
# Убить
kill -9 <PID>
# Или использовать другой порт
PORT=4322 npm start

Database locked

SQLite locked (много write-запросов):

PRAGMA journal_mode=WAL;
PRAGMA busy_timeout=5000;

Или перейти на PostgreSQL (через emdash/db/postgres).

FAQ

Q: Можно ли деплоить без Docker?
A: Да, просто Node.js на VPS: npm ci, npm run build, npm start + pm2.

Q: Где хранить data.db?
A: В volume (Docker) или отдельной папке /var/lib/rksh/. Не в контейнере (исчезнет при пересборке).

Q: Как обновить EmDash версию?
A: npm update emdash, проверить compatibility, npm run build, протестировать staging.

Q: Сколько стоит хостинг?
A: VPS 2 GB RAM, 1 vCPU ~ $5/mo (DigitalOcean, Hetzner). + storage (~$0.05/GB).
