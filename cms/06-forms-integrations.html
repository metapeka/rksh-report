Формы и интеграции

Обзор

Формы на сайте РКШ используются для:

  • Записи на экскурсию/представление
  • Заявки на обучение
  • Консультаций
  • Отзывов

Требование: отправка в email и CRM. Пока нет реализации, только архитектура.

Типы форм

1. Форма записи на экскурсию

Расположение: главная страница (CTA «Получить презентацию»), страница Контакты

Поля:

Поле Тип Обязательное Описание
parent_name string ✅ Имя родителя
student_name string ❌ Имя ребёнка (или "Родитель" для курсов)
phone string ✅ Телефон (+7...)
email email ❌ Email для связи
course select ❌ Выбранная программа («1 класс», «Шахматы»)
message text ❌ Комментарий/вопрос
consent boolean ✅ Согласие на обработку ПДн

Действие: при отправке → email на info@rksh-msk.ru + запись в CRM (Bitrix24)

2. Форма заявки на обучение

Расположение: страницы Цены, страницы курсов

Поля:

  • parent_name (string, required)
  • phone (string, required)
  • email (email)
  • child_birth_year (number) — год рождения ребёнка
  • program (string) — выбранная программа
  • comment (text)

Отправка: email + CRM → автоматическое создание лида

3. Форма консультации

Расположение: страница Контакты

Поля:

  • name (string)
  • email (email, required)
  • subject (string) — тема вопроса
  • message (textarea, required)

4. Форма отзыва

Расположение: страница Отзывы

Поля:

  • author_name (string, required)
  • author_photo (image) — необязательно
  • rating (number 1-5) — оценка
  • content (portableText) — текст отзыва
  • children_grade (string) — класс ребёнка

Модерация: перед публикацией проходит проверку администратором

Интеграция с CRM

Bitrix24

Webhook: https://<domain>.bitrix24.ru/rest/1/<token>/crm.lead.add.json

Поля:

{
  "fields": {
    "TITLE": "Заявка с сайта РКШ",
    "NAME": "{parent_name}",

POST https://<domain>.amocrm.ru/api/v4/leads

Валидация форм

Клиентская:

<input type="text" name="phone" pattern="[+][0-9]{10,15}" required />
<input type="email" name="email" required />

Серверная:

if (!data.phone || !/^\+?[0-9]{10,15}$/.test(data.phone)) {
  return new Response(JSON.stringify({ error: "Invalid phone" }), { status: 400 });
}

Защита от спама

Сроки:

  1. CAPTCHA — добавить потом (reCAPTCHA v3 или hCaptcha)
  2. Honeypot — скрытое поле, которое должен остаться пустым
  3. Rate limiting — не более 3 отправок с IP в минуту

Honeypot пример:

<input type="text" name="website" style="display:none" />

Если website не пустое → игнорировать отправку (бот).

Хранение заявок

Коллекция: applications (можно создать)

Поля:

Поле Тип Описание
form_type string contact, enrollment, review
data json Все поля формы как объект
submitted_at datetime Дата отправки
ip_address string IP пользователя (анонимизировать)
processed boolean Обработана ли заявка

Использование: можно просматривать в админке → Контент → Applications

Webhook-интеграция

Общий подход:

const webhookUrl = EMV_WEBHOOK_URL; // из .env
await fetch(webhookUrl, {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify(payload),

Retry logic: при ошибке 5xx — повторить через 5 сек, максимум 3 попытки

Пример компонента формы (Astro)

---
// src/components/ContactForm.astro
interface Props {
  title?: string;
  course?: string; // предвыбранный курс
}

const { title = "Оставить заявку", course = "" } = Astro.props;

if (Astro.request.method === "POST") {
  // обработка отправки
  const formData = await Astro.request.formData();
  const data = Object.fromEntries(formData);
  // отправка на API
  const res = await fetch("/_emdash/api/forms/submit", {
    method: "POST",
    body: JSON.stringify(data),
    headers: { "Content-Type": "application/json" },
  });
  if (res.ok) {
    // успех
  } else {
    // ошибка
  }
}

Тестирование форм

  1. Заполнить все поля корректно → отправить
  2. Проверить email (входящие)
  3. Проверить CRM (создался ли лид)
  4. Проверить, что дубли не создаются (同一个 номер/IP)
  5. Проверить, что пустые обязательные поля блокируют отправку
