Миграция контента с rksh-msk.ru

Обзор

Полный перенос контента с сайта https://rksh-msk.ru (WordPress) на EmDash CMS.

Объём: 38 страниц, сотни изображений, видео, документы.

Источник контента: docs/content-summary.md — уже собранный текст всех страниц.

Этапы миграции

✅ Сделано:

  • Спаршены все 38 страниц
  • Определена структура меню
  • Загружены медиафайлы (лого, фото)
  • Сопоставлены URL

Результат: docs/site-structure.md, docs/content-summary.md

✅ Сделано:

Задача: Добавить новые коллекции (teachers, schedule, advantages, media, courses_*, prices, reviews, sveden_sections)

Этап 3: Перенос контента

Способ 1: Ручной (для 38 страниц — приемлемо)

  1. Открыть админку /_emdash/admin
  2. Для каждой страницы (согласно content-summary.md):
    • Контент → Страницы → Создать
    • Заполнить title, slug
    • Вставить контент из content-summary.md
    • Использовать блоки из @rksh/blocks где возможно
    • Сохранить → Опубликовать

Порядок страниц (приоритет):

  1. Главная (home) — базовая страница
  2. Страницы меню: О школе, Цены, Контакты, Отзывы
  3. Уровни школы: 0 класс, начальная, средняя
  4. Курсы (дошкольники, школьники)
  5. Дополнительные: Расписание, Преподаватели, Преимущества, Медиа
  6. Сведения, Политика

Способ 2: Автоматический (скриптом)

Написать Node.js скрипт scripts/import-content.js:

import { createClient } from "emdash";
import content from "../docs/content-summary.json";

const client = createClient({
  url: "http://localhost:4321",
  token: process.env.EMDASH_TOKEN, // нужно создать API token
});

for (const page of content.pages) {
  await client.entries.create("pages", {
    data: {
      title: page.title,
      slug: page.slug,
      content: page.content, // в формате Portable Text
    },
    status: "published",
  });
}

Требуется: API токен с правами content:create.

Этап 4: Перенос медиа

Изображения

  1. Скачать все изображения со старого сайта:
# WordPress uploads
wget -r -np -nH --cut-dirs=1 \
  -A jpg,jpeg,png,gif,webp,svg \
  https://rksh-msk.ru/wp-content/uploads/
# Сохранить в temp/
  1. Загрузить в EmDash:
    • Через админку (по одному) — долго
    • Через API массово:
// scripts/upload-media.js
const FormData = require('form-data');
const fs = require('fs');
const path = require('path');

async function uploadFile(filepath) {
  const form = new FormData();
  form.append('file', fs.createReadStream(filepath));
  
  const res = await fetch('http://localhost:4321/_emdash/api/media/upload', {
    method: 'POST',
    body: form,
    headers: form.getHeaders(),
  });
  
  const data = await res.json();
  console.log(`Uploaded: ${filepath} → ${data.url}`);
  return data;
}

// Пройти по папке temp/uploads/ и загрузить все
  1. Обновить URL в контенте:
    • Было: https://rksh-msk.ru/wp-content/uploads/2022/07/logo1-473.png
    • Стало: /_emdash/api/media/file/<id>
    • Нужен скрипт для замены:
-- Вручную через SQL (опасно!)
UPDATE entries 
SET content = replace(content, 'https://rksh-msk.ru/wp-content/', '/_emdash/api/media/file/')
WHERE collection='pages';

Лучше: при импорте контента сразу подставлять правильные URL.

Видео

  1. Видео с сайта — скачивать не нужно (большие файлы). Лучше:

    • Если уже на YouTube — оставить embed-код
    • Если локальные — загрузить через админку (лимиты!)

  2. Включить в media коллекцию тип video, хранить YouTube ID или файл.

Документы (PDF)

  • Скачать из wp-content/uploads/ (политика, лицензии)
  • Загрузить в EmDash через API
  • Заполнить documents коллекцию

Implementation: src/middleware.ts:

export const onRequest = async (context) => {
  const { url } = context;
  const path = url.pathname;
  
  const redirects: Record<string, string> = {
    // если старые URL отличались
    "/about": "/o-shkole/",
  };
  
  if (redirects[path]) {
    return Response.redirect(new URL(redirects[path], url), 301);
  }
};

Автоматизация миграции

Скрипт scripts/migrate.js

#!/usr/bin/env node

import fs from "fs";
import path from "path";
import { createClient } from "emdash";

const client = createClient({
  url: "http://localhost:4321",
  token: process.env.EMDASH_TOKEN,
});

const content = JSON.parse(fs.readFileSync("../docs/content-summary.json"));

async function migrate() {
  for (const page of content.pages) {
    try {
      await client.entries.create("pages", {
        data: {
          title: page.title,
          slug: page.slug,
          content: parsePortableText(page.content), // конвертация HTML → Portable Text
        },
        status: "published",
      });
      console.log(`✅ ${page.title}`);
    } catch (err) {
      console.error(`❌ ${page.title}:`, err.message);
    }
  }
}

migrate();

Запуск:

EMDASH_TOKEN=xxx node scripts/migrate.js

Конвертация HTML → Portable Text

Текст в content-summary.md — plain HTML? Нужно преобразовать в Portable Text JSON.

Пример:

<h2>Преимущества</h2>
<ul>
  <li>Маленькие классы</li>
  <li>Чуткие учителя</li>
</ul>

→ Portable Text:

[
  { "_type": "block", "style": "h2", "children": [{ "_type": "span", "text": "Преимущества" }] },
  { "_type": "list", "listItem": "number", "children": [
    { "_type": "span", "text": "Маленькие классы" },
    { "_type": "span", "text": "Чуткие учителя" }
  ]}
]

Инструмент: html-to-portabletext пакет или написать самому.

Сложные случаи

Галерея изображений

На старом сайте: галерея на JavaScript (возможно, галерея-плагин).
В EmDash: нужно создать media коллекцию с type: "gallery" и массивом items.

Решение:

  1. Собрать все URL изображений галереи
  2. Создать запись в media:
{
  "type": "gallery",
  "title": "Фотогалерея школы",
  "items": [
    { "url": "photo1.jpg", "caption": "Корпус" },
    { "url": "photo2.jpg", "caption": "Учебный класс" }
  ]
}
  1. На странице /fotogalereya/ отрендерить компонент Gallery, который берёт эту запись.

Формы (контакты)

На старом сайте: форма отправлялась на PHP-скрипт.
В EmDash: нужно написать плагин форм (см. 06-forms-integrations.md).

Временное решение: простую форму на Netlify Forms или Formspree, пока своя не готова.

Видеоархив

Старый сайт: возможно, YouTube-embed или локальные файлы.
Нужно:

  1. Найти все YouTube ID
  2. Создать media записи с type: "video"
  3. На странице /videoarhiv/ — список видео (компонент VideoList)

Отзывы

Стандартные текстовые отзывы (сейчас пусто).
После создания reviews коллекции — добавить через админку вручную или импорт.

Откат (rollback)

Если миграция сломала что-то:

  1. Бэкап БД: уже сделан (data.db.backup)
  2. Восстановить:
cp data.db.backup data.db
# перезапустить сервер
  1. Медиа: вернуть папку uploads/ из бэкапа

Важно: делать бэкап перед каждой большой миграцией.

Частые проблемы миграции

Контент потерял форматирование

Причина: HTML → Portable Text конвертация простая, теги <strong> не стали bold.

Решение: улучшить парсер HTML→Portable Text, или ручная правка.

Изображения битые

Проверить:

find uploads/ -type f -size 0  # пустые файлы
grep -r "broken-image-url" src/

Slug конфликты

Problem: два разных WordPress-поста получили одинаковый slug после транслитерации.

Solution: добавить цифру (o-shkole-2), обновить меню.

Проверка целостности

Список всех страниц

SELECT id, title, status FROM entries WHERE collection='pages' ORDER BY id;

Количество медиафайлов

find uploads/ -type f | wc -l
